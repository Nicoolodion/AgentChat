1. DNS - Theoretische Grundlagen

1.1 Was ist DNS?

Definition: Domain Name System (DNS)

Das Domain Name System (DNS) ist ein hierarchisches, dezentralisiertes Namenssystem fuer Computer, Dienste und andere Ressourcen, die mit dem Internet oder einem privaten Netzwerk verbunden sind. Es uebersetzt menschenlesbare Domainnamen (wie www.example.com) in IP-Adressen (wie 192.168.1.1 oder 2001:db8::1), die fuer die Netzwerkkommunikation erforderlich sind.

Das DNS ist einer der fundamentalen Bausteine des Internets und funktioniert als das "Telefonbuch des Internets". Ohne DNS muessten Benutzer sich komplexe numerische IP-Adressen merken, um auf Websites zuzugreifen. Das System wurde 1983 von Paul Mockapetris entwickelt und hat sich seitdem zu einem kritischen Infrastrukturelement entwickelt.

Kernfunktionen des DNS

  • Namensaufloesung: Umwandlung von Domainnamen in IP-Adressen
  • Reverse-DNS: Umwandlung von IP-Adressen in Domainnamen
  • Lastverteilung: Verteilung von Anfragen auf mehrere Server
  • Mail-Routing: Steuerung von E-Mail-Zustellung durch MX-Records
  • Service-Discovery: Auffinden von Diensten im Netzwerk

1.2 DNS-Aufloesung

Der Prozess der DNS-Aufloesung umfasst mehrere Schritte und involviert verschiedene Server-Typen. Bei einer typischen Anfrage durchlaeuft das System folgende Phasen:

Phasen der DNS-Aufloesung
Phase Beschreibung Beteiligter Server
1. Cache-Pruefung Pruefung des lokalen DNS-Caches Resolver/Client
2. Rekursive Anfrage Anfrage an den rekursiven DNS-Resolver DNS-Resolver
3. Root-Server Anfrage an Root-Nameserver Root-Server (13 weltweit)
4. TLD-Server Weiterleitung an TLD-Nameserver TLD-Server (.com, .de, etc.)
5. Autoritaerer Server Anfrage beim autoritaeren Nameserver Domain-Nameserver
6. Antwort Rueckgabe der IP-Adresse Resolver an Client

DNS-Cache

DNS-Informationen werden auf verschiedenen Ebenen zwischengespeichert, um die Effizienz zu steigern und die Serverlast zu reduzieren:

  • Browser-Cache: Speichert DNS-Eintraege im Webbrowser
  • Betriebssystem-Cache: Systemweiter DNS-Cache
  • Resolver-Cache: Cache des DNS-Resolvers (oft beim ISP)
  • CDN-Cache: Content Delivery Network Caching

Die Lebensdauer von Cache-Eintraegen wird durch den TTL-Wert (Time To Live) im DNS-Record bestimmt. Nach Ablauf der TTL wird der Eintrag neu abgefragt.

1.3 DNS-Record-Typen

DNS-Records speichern verschiedene Arten von Informationen. Die wichtigsten Record-Typen sind:

Wichtige DNS-Record-Typen
Record-Typ Zweck Beispiel
A IPv4-Adresse zuweisen example.com -> 93.184.216.34
AAAA IPv6-Adresse zuweisen example.com -> 2606:2800:220:1::
CNAME Alias/Weiterleitung www.example.com -> example.com
MX Mail-Server festlegen mail.example.com (Prioritaet 10)
NS Nameserver definieren ns1.example.com, ns2.example.com
TXT Textinformationen SPF, DKIM, Verifikation
PTR Reverse-DNS 34.216.184.93.in-addr.arpa
SOA Start of Authority Zonen-Administrationsdaten

1.4 DNS-Server-Typen

Autoritaere Nameserver

Autoritaere Nameserver enthalten die tatsaechlichen DNS-Records fuer eine Domain. Sie sind die ultimative Quelle der Wahrheit fuer DNS-Informationen einer bestimmten Zone. Jede Domain muss mindestens zwei autoritaere Nameserver haben (Primaer und Sekundaer).

Rekursive Resolver

Rekursive Resolver (auch Caching-Nameserver genannt) beantworten DNS-Anfragen von Clients, indem sie die hierarchische DNS-Struktur durchlaufen. Sie speichern die Ergebnisse zwischen, um zukuenftige Anfragen schneller zu beantworten.

Forwarding Server

Forwarding-Server leiten alle Anfragen an einen anderen DNS-Server weiter, anstatt selbst die Aufloesung durchzufuehren. Dies wird oft in Unternehmensnetzwerken verwendet, um zentrale Kontrolle und Filterung zu ermoeglichen.

DNS-Filter und Ad-Blocker

Moderne DNS-Server wie AdGuard, Pi-hole oder NextDNS bieten zusaetzliche Funktionen:

  • Werbeblockierung: Blockieren von Werbe- und Tracking-Domains
  • Malware-Schutz: Filterung boesartiger Domains
  • Kindersicherung: Sperrung unangemessener Inhalte
  • DoH/DoT: Verschluesselte DNS-Anfragen (DNS over HTTPS/TLS)

2. AdGuard DNS-Server Implementierung

2.1 Ueberblick ueber AdGuard

AdGuard Home ist ein netzwerkweiter Werbeblocker, der als DNS-Server fungiert. Im Gegensatz zu browserbasierten Ad-Blockern filtert AdGuard den Datenverkehr fuer alle Geraete im Netzwerk, ohne dass auf jedem Geraet Software installiert werden muss.

Hauptfunktionen von AdGuard

  • Netzwerkweite Werbe- und Tracker-Blockierung
  • Schutz vor Phishing und Malware-Domains
  • Elternkontrolle und Content-Filterung
  • Detaillierte Statistiken und Query-Logs
  • Unterstuetzung fuer DNS-over-HTTPS (DoH) und DNS-over-TLS (DoT)
  • Eigene Filterlisten und Whitelist-Konfiguration
  • Benutzerfreundliches Webinterface

2.2 Docker Compose Konfiguration

Fuer die Implementierung wurde AdGuard Home als Docker-Container auf einem Homeserver bereitgestellt. Docker Compose bietet eine einfache und reproduzierbare Methode zur Bereitstellung.

Schritt 1: Docker Compose Datei erstellen

Die folgende docker-compose.yml Datei wurde fuer die AdGuard-Installation verwendet:

version: "3"
services:
  adguardhome:
    image: adguard/adguardhome:latest
    container_name: adguardhome
    ports:
      - 53:53/tcp
      - 53:53/udp
      - 784:784/udp
      - 853:853/tcp
      - 3000:3000/tcp
      - 8040:80/tcp
    volumes:
      - ./workdir:/opt/adguardhome/work
      - ./confdir:/opt/adguardhome/conf
    restart: unless-stopped

Port-Erklaerung

Verwendete Ports in der Docker-Konfiguration
Port Protokoll Verwendung
53 TCP/UDP Standard DNS-Port
784 UDP DNS-over-QUIC
853 TCP DNS-over-TLS
3000 TCP Erstkonfiguration (Setup-Wizard)
8040 TCP Webinterface fuer Administration

2.3 Installation und Einrichtung

Schritt 2: Container starten und Erstkonfiguration

Nach dem Start des Containers mit docker-compose up -d wird das Webinterface ueber Port 3000 fuer die Erstkonfiguration aufgerufen. Der Setup-Wizard fuehrt durch die grundlegende Konfiguration:

  • Admin-Benutzername und Passwort festlegen
  • DNS-Ports konfigurieren (hier: 53 fuer DNS)
  • Webinterface-Port festlegen (hier: 8040)
Schritt 3: DNS-Server Einstellungen

Im naechsten Schritt werden die Upstream-DNS-Server konfiguriert. AdGuard leitet Anfragen fuer nicht blockierte Domains an diese Server weiter. Empfohlene Optionen sind:

  • Cloudflare: 1.1.1.1 und 1.0.0.1 (schnell, datenschutzfreundlich)
  • Quad9: 9.9.9.9 (mit Malware-Schutz)
  • Google: 8.8.8.8 und 8.8.4.4 (sehr zuverlaessig)
Schritt 4: Filterlisten konfigurieren

AdGuard verwendet Filterlisten, um Werbung, Tracker und unerwuenschte Inhalte zu blockieren. Standardmaessig sind mehrere Listen aktiviert:

  • AdGuard DNS Filter
  • AdAway Default Blocklist
  • MalwareDomainList.com Hosts List

Zusaetzliche Listen koennen nach Bedarf hinzugefuegt werden.

2.4 Windows Client-Konfiguration

Schritt 5: Windows DNS-Einstellungen anpassen

Um den Windows-Client fuer die Verwendung des AdGuard DNS-Servers zu konfigurieren, muessen die DNS-Einstellungen der Netzwerkverbindung angepasst werden:

  1. Netzwerk- und Freigabecenter oeffnen
  2. Aktive Netzwerkverbindung auswaehlen
  3. Eigenschaften -> Internetprotokoll Version 4 (TCP/IPv4)
  4. "Folgende DNS-Serveradressen verwenden" auswaehlen
  5. IP-Adresse des AdGuard-Servers als bevorzugten DNS-Server eingeben

2.5 Funktionstest und Ergebnisse

Schritt 6: Funktionsueberpruefung

Nach der Konfiguration wird die Funktionalitaet durch Browsen verschiedener Websites und durch Ping-Tests ueberprueft. Das AdGuard Dashboard zeigt detaillierte Statistiken:

  • Anzahl der verarbeiteten DNS-Anfragen
  • Anzahl der blockierten Anfragen
  • Top blockierte Domains
  • Top angefragte Domains
  • Query-Logs fuer detaillierte Analyse

Das Dashboard zeigt, dass AdGuard erfolgreich DNS-Anfragen verarbeitet und Werbe- sowie Tracking-Domains effektiv blockiert. Die blockierten Anfragen werden in Echtzeit angezeigt, was eine transparente Kontrolle ermoeglicht.

3. Persoenliche Stellungnahme

3.1 Uebersicht und Funktionsumfang

AdGuard Home bietet einen umfassenden Funktionsumfang fuer ein netzwerkweites DNS-Filtering. Die wichtigsten Staerken sind:

Funktionsbewertung

Werbeblockierung: Hervorragend
Tracker-Schutz: Hervorragend
Malware-Schutz: Gut
Elternkontrolle: Gut
Statistiken: Sehr gut
DoH/DoT Support: Hervorragend

Im Vergleich zu Alternativen wie Pi-hole bietet AdGuard eine modernere Benutzeroberflaeche und integrierte Verschluesselungsoptionen (DoH/DoT) ohne zusaetzliche Konfiguration. Die vorinstallierten Filterlisten decken die meisten Anwendungsfaelle ab, koennen aber bei Bedarf erweitert werden.

3.2 Einfachheit der Konfiguration

Die Docker-basierte Installation erheblich vereinfacht die Bereitstellung. Die wichtigsten Vorteile der Docker-Implementierung:

Bewertung der Konfiguration
Aspekt Bewertung Anmerkung
Installation Sehr einfach Docker Compose startet alles automatisch
Erstkonfiguration Einfach Setup-Wizard fuehrt durch alle Schritte
Port-Konfiguration Mittel Port 53 erfordert ggf. root-Rechte
Filterlisten Sehr einfach Vorkonfiguriert, einfach anpassbar
Client-Konfiguration Einfach DNS-Einstellung auf Client-Geraeten
Wartung Sehr einfach Automatische Updates via Docker

Die Konfiguration ueber das Webinterface ist intuitiv und gut strukturiert. Besonders hilfreich ist die Moeglichkeit, Einstellungen ohne Neustart des Dienstes zu aendern. Die Query-Logs bieten wertvolle Einblicke in das Netzwerkverhalten.

Herausforderungen

  • Port 53: Auf einigen Systemen ist Port 53 bereits durch systemd-resolved belegt und muss freigegeben werden
  • IPv6: Die Konfiguration fuer IPv6-Netzwerke erfordert zusaetzliche Aufmerksamkeit
  • Router-Konfiguration: Fuer netzwerkweite Nutzung muss der Router auf den AdGuard-Server zeigen

3.3 Fazit und Empfehlung

AdGuard Home ist eine ausgezeichnete Wahl fuer alle, die netzwerkweite Werbeblockierung und verbesserte Privatsphaere suchen. Die Docker-basierte Installation auf einem Homeserver bietet maximale Flexibilitaet und einfache Wartung.

Gesamtbewertung

Gesamtnote: 9/10

AdGuard Home ueberzeugt durch einfache Installation, umfangreiche Funktionen und eine moderne Benutzeroberflaeche. Besonders fuer Docker-erfahrene Nutzer ist die Einrichtung im Homeserver-Umfeld optimal geeignet. Die Kombination aus Werbeblockierung, Tracker-Schutz und verschluesselten DNS-Anfragen bietet einen deutlichen Mehrwert fuer die gesamte Netzwerkinfrastruktur.

Empfohlene Einsatzszenarien

  • Homeserver-Umgebungen: Ideale Ergaenzung fuer bestehende Docker-Setup
  • Smart-Home-Netzwerke: Schutz von IoT-Geraeten vor Tracking
  • Familiennetzwerke: Elternkontrolle und kindersicheres Internet
  • Privacy-fokussierte Nutzer: DoH/DoT fuer verschluesselte DNS-Anfragen

Die Entscheidung fuer AdGuard gegenueber Alternativen wie Pi-hole oder BIND mit Blocklisten ist durch die moderne Oberflaeche und die integrierte Verschluesselungsunterstuetzung gerechtfertigt. Fuer reine DNS-Aufloesung ohne Filterfunktionen waeren BIND oder dnsmasq leichtgewichtigere Alternativen.
